System Architecture

Data Flow & Security Architecture

Understand how the secure gateway, multi-tenant databases, and ML models interact to protect user data and classify threats.

🔒 1. Gateway & Anti-Spoofing

The launcher gateway on port 8000 intercepts all incoming requests to FraudGuard and SpamShield, enforcing critical network security boundaries:

  • Header Sanitization: Strips user-supplied header tokens (such as X-User-Id, X-User-Role) to block spoofing attempts.
  • Verified Injection: Securely decodes backend Flask sessions and injects verified user identity context into proxied requests.
  • Session Restrictions: Blocks unauthorized anonymous access to any proxied folders or analytical sub-endpoints.

💾 2. Multi-Tenant Data Isolation

Privacy is maintained by isolating all system entities at the database, session, and file storage level:

  • Database Partitioning: Every processed anomaly record, blocked item, model metric, and alert log is explicitly tagged with the owner's User ID.
  • Dashboard Separation: Queries filter datasets dynamically. A standard user sees a completely clean dashboard on first login, with zero data leakage.
  • Folder Sandboxing: Uploaded transaction files are segregated under user-specific subfolders. Cross-user access attempts return a 403 Forbidden response.

🧠 3. Dual ML Ensemble Engine

Two independent machine learning configurations run concurrently to process text and tabular transaction records:

  • Unsupervised Fraud Engine: FraudGuard applies an ensemble of Isolation Forest, One-Class SVM, and DBSCAN to detect transaction anomalies.
  • Supervised Spam Engine: SpamShield uses TF-IDF and a voting ensemble (Random Forest, SVM, LR) to classify messages, with model retraining.
  • Central Alerts Ledger: Incidents from both engines are securely hashed and stored in a unified MySQL database on the gateway.

🌐 System Topology & Header-Injection Architecture

Interactive Architecture Flow: Requests hitting port 8000 have their authentication checked. The gateway then strips all spoofed credentials and injects verified identity headers (X-User-Id, X-User-Role) before reverse-proxying requests to internal microservices on ports 5000 and 5001.

🧠 Advanced Machine Learning Pipelines

Unsupervised Anomaly Scoring Pipeline

FraudGuard performs zero-configuration, zero-label analysis on uploaded financial ledger sheets (CSV/Excel). The pipeline runs in four stages:

  1. Preprocessing & Scaling: Non-numeric parameters are automatically trimmed. Variables are scaled using RobustScaler to minimize skew from actual outliers.
  2. Dynamic Ensemble Execution:
    • Isolation Forest: Tree isolation partition-path counts determine initial anomaly scores.
    • One-Class SVM: A radial basis function (RBF) kernel projects points to detect boundary anomalies.
    • DBSCAN: Spatial clustering groups normal clusters and identifies scattered outlier records.
  3. Large Dataset Handling (Safety Trigger): For large uploads (N > 3000), the server automatically disables the slower, quadratic complexity algorithms (DBSCAN & One-Class SVM) to prevent gateway timeouts, relying on fast forest isolation.
  4. Alert Ledger Propagation: Verified outliers trigger automatic SHA-256 alerts injected directly into the central SentinelLedger database.
Ensemble Formula & Scoring

The aggregate fraud score is calculated by combining normalized scores from all active classifiers:

Score = w₁·S_IF + w₂·S_OCSVM + w₃·S_DBSCAN

If the combined probability exceeds 0.65, the record is immediately flagged as fraudulent, hashed into the blockchain audit ledger, and sent to the alerts table.

Supervised Voting Ensemble Pipeline

SpamShield analyzes emails, text logs, and bulk text datasets using a pre-trained feature vocabulary:

  1. TF-IDF Vectorization: Raw message content is converted into dynamic token frequencies based on term frequency-inverse document frequency weights, excluding common English stop words.
  2. Linguistic Feature Extraction: Extracts meta-level features such as capital letter ratios, link density, special character frequencies, and sentiment analysis scores.
  3. Voting Classifier Ensemble:
    • Logistic Regression: Fast linear probabilistic baseline.
    • Random Forest: Non-linear tree ensemble to catch compound keyword occurrences.
    • Support Vector Machine (SVC): Maximizes separation margin for complex text classes.
  4. Incremental Model Retraining: Analysts with appropriate permissions can submit newly labeled files to retrain models, shifting the TF-IDF weights to adapt to fresh spam patterns.
Ensemble Voting Method

The final spam decision is determined using soft voting, taking the average of prediction probabilities:

P(Spam) = [P_LR(S) + P_RF(S) + P_SVC(S)] / 3

Soft voting provides higher robustness than hard majority voting, as it considers the confidence levels of each individual model before rendering a classification verdict.

👥 Role-Based Authorization & Isolation Policy

The system enforces user segregation dynamically at the API level based on session variables. Here is how permissions map across roles:

Role Scope of Visible Data File Storage Access Model Retraining Audit Access
analyst Owner-only data (segregated by user_id context). Own folder only (uploads/{user_id}/). Blocked (Returns 403 / Warning). Own logs only.
admin Global visibility (sees all records across all tenants). Global read/write across all user sandboxes. Authorized (Retrain ensemble models). Full action logs & session states.
auditor Read-only access to all transaction & classification logs. Read-only access to compiled datasets. Read-only view. Authorized (Full blockchain log integrity checks).

🔗 Tamper-Proof Audit Ledger (Blockchain Integration)

Whenever a suspicious transaction is flagged by FraudGuard, the alert is logged using a hash-linked data structure inspired by blockchain principles. This prevents unauthorized historical modifications of security incidents:

BLOCK #1 (Genesis)
Prev Hash: genesis
Data: ID: tx_101, Score: 0.82
Hash: 1948572...
➡️
BLOCK #2
Prev Hash: 1948572...
Data: ID: tx_102, Score: 0.94
Hash: 9082341...
➡️
BLOCK #3
Prev Hash: 9082341...
Data: ID: tx_103, Score: 0.71
Hash: 3459812...
  • Linear Cryptographic Chain: Each block hashes its own transaction parameters together with the previous_hash string.
  • Tamper Detection: Changing even a single amount or score value in SQLite retroactively invalidates all downstream hash chains, alerting security auditors instantly.
  • Auditor Verification: Users with the auditor role have access to verification endpoints that re-run hash checking logic to guarantee integrity.

Purpose & Technology

Both projects are fully independent — no shared backend, no merged logic.

📊 FraudGuard

Detects anomalous financial transactions using unsupervised machine learning. Designed for analysts who need to upload raw transaction data and immediately identify high-risk patterns without labelled training data.

  • Ensemble of 3 unsupervised ML models
  • SQLite with blockchain-style audit log
  • Real-time monitoring thread
  • Power BI CSV/JSON export

🛡️ SpamShield

Classifies emails as spam or ham using supervised machine learning trained on labelled datasets. Supports single message input, file upload, and bulk CSV processing with per-row predictions.

  • Django MVC backend
  • Voting ensemble (RF + LR + SVM)
  • TF-IDF + hand-crafted feature engineering
  • Alert system with email/webhook notifications
  • Model retraining endpoint

Getting Started

How to Run Both Projects

Run each project in a separate terminal window. Both must be running to use the preview buttons.

1

Run FraudGuard (Port 5000)

Open a terminal, navigate to the pbi card folder and start Flask.

cd "pbi card" pip install -r requirements.txt python app.py --port 5000
2

Run SpamShield (Port 5001)

Open a second terminal, navigate to the spam detection folder and start Django.

cd Spam-Detection-Classifier-main\Spam-Detection-Classifier-main\spam_detection pip install -r requirements.txt python manage.py runserver 5001 --noreload
3

Open This Page

Simply open this index.html file directly in your browser — no server needed.

cd launcher python serve.py

Then open: http://localhost:8000

4

One-Click Launch (Windows)

Use the batch script to start all apps at once from the project root.

run_all.bat

Opens both servers and this page automatically.
